一、设计规则
从原则推导，规范从规则推导
R1
目录区只做一件事：帮用户找到文件
操作按钮、搜索、锁状态都服务于这一目标。任何不帮助用户找到文件的元素都不应该出现。
R2
常驻操作区只放高频操作
顶部常驻：新增、定位、收起全部、收起面板。低频操作（重命名、删除、复制路径）放右键菜单，不占常驻空间。
R3
锁状态只展示当前用户是否持有
持有锁：节点右侧 colorPrimary 小锁图标。未持有：不展示任何图标，不占空间。不展示是谁持有，避免信息噪音。
R4
内联创建，不弹 Modal
点击新增后展示下拉选择文件类型，选择后在当前目录下插入内联输入框。回车确认，ESC 或失焦取消。
R5
搜索支持 @ 指令快速过滤
普通输入按文件名模糊过滤，@ 指令按元数据过滤。过滤结果展示匹配数量提示，搜索框激活态用主色区分。
R6
节点 hover 才显示操作按钮
操作按钮（···）默认不可见，hover 才出现，减少视觉噪音。节点激活态用 colorBgActive，文字不变（焦点稳定原则）。
